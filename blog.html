What is the difference between map(), filter(), forEach() and find() ?

In JavaScript map() returns a new array with updated elements from calling a function for every single array elements. It does not change original array and it doesn't execute for empty array.

The filter() method creates a new array for all the elements of the array which satisfy certain conditions.

The forEach() method calls a function and iterates over the elements of an array.The forEach() method is used we dont need anything in return.

The find() method executes a function for each array element and returns only the value of the first element that satisfy certain conditions. The find() method returns undefined if no elements are found.

What is the difference between local storage, session storge and cookies in javascript?

Local storage is used to store data that persist even after the browser is closed. Its only can accessed by client. Data stored in local storage is not sent to the server.

Session storage is used to store data but these data does not persist after the browser is close. So when session is finish, data will be deleted. As like local storage its only can accessed by client and stored data is not sent to the server.

Cookies are small text files stored in the user's browser.These are used to store data that needs to be sent to the server with each HTTP request. Cookies can be accessed by both server and clients.

What is EventLoop and how it works?

In javascript, the Event Loop is a mechanism that allows for efficient and responsive handling of events in a single-threaded environment by managing the order of execution of code in response to events and callbacks.How the Event Loop works given below:

An event handler function is added to a queue when an event is triggered. New event handler functions are continuously checked for by the event loop. If a function is discovered, it is run. The loop waits for new events if there are no functions in the queue. New events or callbacks may be generated when an event handler code is called. After the current function has finished running, these new callbacks are added to a queue and will be performed in the order they were added. As long as there are events or callbacks in the queue, the event loop keeps going. The loop ends when the queue is empty and doesn't restart until new events are triggered.
